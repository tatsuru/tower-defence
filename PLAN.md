# 実装計画

要件定義: [REQUIREMENTS.md](./REQUIREMENTS.md)

## フェーズ構成

```
Phase 1: プロジェクトセットアップ
Phase 2: グリッドとマップ生成
Phase 3: タワー設置
Phase 4: 敵の移動
Phase 5: ウェーブシステム
Phase 6: 戦闘
Phase 7: タワーアップグレード
Phase 8: UI・ゲーム状態管理
```

依存関係:
- Phase 2 → Phase 3, 4
- Phase 3 + Phase 4 → Phase 5
- Phase 3 + Phase 4 → Phase 6
- Phase 3 → Phase 7
- 全フェーズ → Phase 8

---

## Phase 1: プロジェクトセットアップ

### Task 1-1: Vite + TypeScript + Phaser.js の初期化

- `npm create vite` で TypeScript テンプレートを作成
- `phaser` パッケージをインストール
- 不要なボイラープレートを削除し、Phaser の `Game` インスタンスを起動するだけの最小構成にする
- ブラウザで空のキャンバスが表示されることを確認

---

## Phase 2: グリッドとマップ生成

### Task 2-1: グリッドの描画

- 20×15 のグリッドを Phaser の Graphics で描画
- セルの種類（通路・設置可能・設置不可）を色で表現
  - 設置可能: 緑系
  - 通路（経路）: 茶色系
  - 設置不可: グレー

### Task 2-2: ランダムマップ生成

- `MapGenerator` クラスを実装
- グリッド上にランダムな地形（設置不可マス）を配置
- 入口（左端）・出口（右端）をランダムな行に決定
- 生成結果を `Cell[][]` の2次元配列として返す

### Task 2-3: 固定経路の生成

- 入口から出口まで BFS で最短経路を1本求める
- 経路上のセルを「通路」として確定し、タワー設置不可にする
- 経路が存在しない場合はマップを再生成する（最大リトライ回数を設ける）
- 経路を視覚的に描画する

---

## Phase 3: タワー設置

### Task 3-1: タワーのデータ定義

- `src/data/towers.ts` にタワー定義を作成
- 5種類（弓兵・魔法・砲台・氷・炎）× 3レベルのパラメータを定義
  - ダメージ、攻撃速度、射程、コスト、アップグレードコスト、外見（色・形）
- データ構造: `TowerDef[]`

### Task 3-2: タワー選択UIパネル

- 画面下部にタワー選択パネルを配置
- 各タワーのボタンにアイコン・名前・コストを表示
- ゴールド不足のタワーをグレーアウト
- クリックで「選択中のタワー」を切り替える

### Task 3-3: タワー設置ロジック

- 設置可能マスをクリックしたときにタワーを設置
- ゴールドを消費する
- 設置不可マス（経路・他のタワーがある・ゴールド不足）への設置を弾く
- 設置後にグリッドの状態を更新

### Task 3-4: タワーの描画（Lv1）

- 設置されたタワーをグリッド上に描画
- タワー種別ごとに異なる色・形で表現（スプライトなしで図形のみ）

---

## Phase 4: 敵の移動

### Task 4-1: 敵のデータ定義

- `src/data/enemies.ts` に敵定義を作成
- 3種類（ゴブリン・オーク・ドラゴン）のパラメータを定義
  - HP、移動速度、撃破報酬、外見（色・サイズ）
- データ構造: `EnemyDef[]`

### Task 4-2: 敵の経路移動と描画

- `Enemy` クラスを実装
- 固定経路のウェイポイントリストを順番に移動する
- HP バーを敵の上部に描画
- 移動速度に従ってフレームごとに位置を更新

### Task 4-3: 拠点到達時の処理

- 敵が出口に到達したときにライフを1減らす
- 敵オブジェクトを破棄する
- ライフが0になったらゲームオーバーフラグを立てる（画面表示は Phase 8 で実装）

---

## Phase 5: ウェーブシステム

### Task 5-1: ウェーブのデータ定義

- `src/data/waves.ts` にウェーブ定義を作成
- 各ウェーブの敵の種類・数・出現間隔を定義
- ウェーブ番号が上がるにつれて数・HPが増えるスケーリング式を実装

### Task 5-2: ウェーブスポーン処理

- `WaveManager` クラスを実装
- 現在のウェーブ定義に従い、時間差で敵を1体ずつスポーンさせる
- ウェーブ内の全敵がスポーンし終えたら「ウェーブ終了待機」状態に入る

### Task 5-3: 準備フェーズ

- 全敵が倒されるか拠点を通過したらカウントダウン（30秒）を開始
- カウントダウンを画面に表示
- 「今すぐ開始」ボタンでカウントダウンをスキップ
- カウントダウン終了で次のウェーブを開始

---

## Phase 6: 戦闘

### Task 6-1: タワーの基本攻撃フレームワーク

- `Tower` クラスに攻撃ロジックの基盤を実装
- 射程内にいる敵を探すクエリ関数
- 攻撃クールダウン管理（攻撃速度に基づくタイマー）
- タワー種別ごとに攻撃処理を差し込めるインターフェースを定義

### Task 6-2: 単体攻撃（弓兵塔・砲台）

- 射程内で最も進んでいる敵（出口に近い敵）を優先して狙う
- ダメージを与える
- 攻撃エフェクト（射線の描画）を簡易実装

### Task 6-3: 範囲攻撃（魔法塔）

- 射程内の全敵にダメージを与える
- 範囲を示すエフェクトを簡易実装

### Task 6-4: スロー効果（氷の塔）

- 対象敵の移動速度を一定時間低下させる
- スロー状態を Enemy に状態フラグとして持たせる

### Task 6-5: 継続ダメージ（炎の塔）

- 対象敵に DoT（一定間隔でダメージ）を付与する
- DoT は複数重ねがけ可・持続時間あり

### Task 6-6: 敵のHP管理と撃破報酬

- HP が0以下になった敵を削除
- 撃破時にゴールドを加算する
- 撃破エフェクト（簡易）を表示

---

## Phase 7: タワーアップグレード

### Task 7-1: タワー詳細UIパネル

- 設置済みタワーをクリックすると詳細パネルを表示
- 現在のレベル・ステータス・アップグレードコストを表示
- Lv3 の場合は「最大レベル」と表示

### Task 7-2: アップグレードロジック

- アップグレードボタンクリックでゴールドを消費
- タワーのステータスをLv2/Lv3のパラメータに切り替える
- ゴールド不足の場合はボタンをグレーアウト

### Task 7-3: レベル別の外見変化

- Lv に応じてタワーの描画（色・サイズ・形）を変える
- レベルアップ時に描画を更新する

---

## Phase 8: UI・ゲーム状態管理

### Task 8-1: ステータスバー

- 画面上部にウェーブ数・ライフ・ゴールドをリアルタイム表示
- 各値が変化したときに即座に反映

### Task 8-2: ゲームオーバー画面

- ライフが0になったらゲームオーバー画面をオーバーレイ表示
- 「生存ウェーブ数（スコア）」を大きく表示
- 「もう一度プレイ」ボタンでゲームをリスタート

### Task 8-3: ゲーム開始・リスタート処理

- タイトル画面 or スタートボタンからゲームを開始
- リスタート時にゲーム状態（タワー・敵・ウェーブ・ゴールド・ライフ）をすべてリセット
- マップをランダム再生成する
